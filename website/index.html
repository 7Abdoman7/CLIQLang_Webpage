What is CLIQLang?

CLIQLang is a quantum programming language designed for defining and executing quantum circuits using an intuitive custom syntax. It bridges the gap between theoretical quantum computing concepts and practical implementation.

Perfect for quantum education, research, and algorithm development, CLIQLang supports standard quantum gates, parametric rotations, custom matrix definitions, oracle operations, conditional logic, and loops.

üéÆ Interactive Commands

Control your quantum circuit execution

  • forward - Apply next gate
  • backward - Revert last gate
  • reset - Reset to |0...0‚ü©
  • run - Execute all gates
  • run <shots> - Execute full circuit from start for N shots; measure all qubits and show a summary table (bitstring, count, probability)
  • rrun - Reset, then execute all gates from the start
  • frun - Execute to final state (no intermediate steps)
  • show <qubits> - Show marginal probabilities over selected qubits (e.g., show 0 or show 0,2). Zero-probability outcomes are omitted.

üìâ Noise & Fidelity

Configure realistic noise models in the interactive setup

  • Depolarization: Probability p to apply a random Pauli (X/Y/Z) to each target after a gate.
  • Gate Fidelity: Average fidelity F. Internally uses a Pauli channel with p = 2¬∑(1 ‚àí F).
  • Decoherence (Amplitude Damping):
    • Rate: Chance per affected qubit after each gate to attempt damping.
    • Decay Probability: If damping triggers, chance that |1‚ü© ‚Üí |0‚ü©.
# When prompted during startup: Noise Depolarization: 0.05 Decoherence Rate: 0.02 Decay Probability: 0.10 Gate Fidelity: 0.995

Quantum Examples

See CLIQLang in action with these quantum algorithms

Bell State Creation

Create quantum entanglement between two qubits

# Create Bell State
H [] [0]          # Superposition
X [0] [1]         # Entanglement
! [0,1]          # Measurement

Quantum Superposition

Put qubits in superposition with Hadamard gates

# Superposition Example
H [] [range(0,3)]  # All qubits in superposition
! [0,1,2]         # Measure all qubits

Oracle Example

Mark specific input states and observe the effect

# Oracle marking states 01 and 11 on two input qubits, flipping output qubit 2
# Prepare superposition on input qubits 0 and 1
H [] [0]
H [] [1]
# Apply oracle (marks decimal states 1 and 3)
ORACLE [0,1] [2] [1,3]
# Optional: amplify marked states (Grover-like step)
H [] [0]
H [] [1]
X [] [0]
X [] [1]
H [] [1]
# Measure all
! [0,1,2]

Running with Noise (Shots)

Configure noise, run multiple shots, and get a summary table

# At interactive startup, enter noise parameters when prompted:
# Noise Depolarization: 0.05
# Decoherence Rate: 0.02
# Decay Probability: 0.10
# Gate Fidelity: 0.995

# Example circuit
H [] [0]
H [] [1]
! [0,1]

# In the interactive shell, run many shots to sample output distribution:
# run 1024
# You will see a table: bitstring, count, probability
